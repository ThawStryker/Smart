# 编写课时脚本

## 触发条件

用户提供课时标题和教学目标，要求编写课时脚本。

## 输入

- 年级（如"三年级"）
- 课程标题
- 教学目标列表

课程大纲从 memory 中读取（USER.md 或独立的大纲文件）。

## 输出格式

**强制使用 4 列表格**：

| 环节 | 任务卡名称 | 资源类型 | 教学逐字稿 |

表格中的 `|` 在逐字稿中需要转义为 `\|`，避免破坏 markdown 表格结构。

资源类型：图片插画 / 视频 / 定制互动 / 互动题板 / 实操工具

---

## 七个固定环节（顺序不可变）

### 1. 封面

- **资源类型**：图片插画
- **内容**：展示课程标题，营造情境氛围
- **任务卡名称**：留空或与主题相关

### 2. 趣味引入

- **资源类型**：视频
- **格式**：小星 × 小宇对话，完整的动画剧情脚本
- **目标**：用小宇的错误认知或生活中遇到的困惑制造冲突，自然引出本课核心问题
- **时长**：1-2 分钟
- **结尾**：必须落在"问题"上，为下一环节的知识讲解埋钩子

### 3. 新知讲解

- **资源类型**：视频 + 定制互动 + 互动题板交替
- **结构**：每个教学目标拆为 1 个或多个子知识点，每个子知识点包含：
  - 视频（小宇提问/误解 → 小星引导 → 图灵鲸讲解）
  - 可选的定制互动（体验式学习）
  - 可选的互动题板（即时反馈）
- **时长**：4-6 分钟（包含互动）
- **核心原则**：新知讲解环节必须覆盖所有教学目标

### 4. 实操体验

- **资源类型**：视频（任务发布）+ 实操工具 + 定制互动
- **结构**：
  - 鲸宝发布任务（视频）
  - 可选的组内讨论环节
  - 实操工具或 Web 互动体验
  - 可选的成果展示/投票环节
- **目标**：将教学目标转化为可操作的任务，学生动手验证所学

### 5. 拓展延伸

- **资源类型**：视频
- **格式**：小宇的疑问 → 小星引导 → 图灵鲸讲解
- **目标**：在核心概念基础上延伸一个"更深一层"的问题（社会影响、伦理讨论、未来展望），不要求掌握，激发好奇
- **时长**：1-2 分钟

### 6. 当堂巩固

- **资源类型**：视频（知识回顾）+ 互动题板（巩固练习）
- **结构**：
  - 视频：总结本课知识点（可用旁白，不需要角色对话）
  - 互动题板：选择题 2-4 题，考试理解而非记忆
- **题目要求**：每道题对应一个教学目标，选项需有明显区分度

### 7. 课后作业

- **资源类型**：图片插画
- **格式**：任务说明（学生视角，语言简单直接）
- **类型**：优先选生活场景中能完成的任务（观察、动手、提问）
- **包含**：任务描述 + 完成标准

## 编剧领域知识

### 三幕叙事结构

趣味引入 → 新知讲解+实操 → 拓展延伸，整体遵循：

```
第一幕（引入）：小宇遇到问题/犯错误 → 制造冲突 → 小星介入
第二幕（探索）：图灵鲸讲解 → 鲸宝发布任务 → 学生动手体验
第三幕（领悟）：拓展延伸 → 回顾总结 → 小宇"哦——原来是这样！"
```

### 钩子设计方法

把枯燥的教学目标翻译成小宇的"问题"：

| 教学目标 | 钩子手法 | 示例 |
|----------|---------|------|
| 文生图的概念 | 小宇画不好画 → 引出 AI 帮忙画 | "你都画不好，AI 没有手怎么画？" |
| 提示词要具体 | 小宇描述太模糊 → AI 画出奇怪东西 | "我说'学校'，AI 画了个平房，我明明想的未来学校！" |
| AI 学习规律 | 小宇觉得 AI 像人一样练习 | "AI 一定是像我一样刻苦练习才学会画画的！" |

核心原则：**知识点不是讲出来的，是被小宇的好奇心"挖"出来的**。

### 对话规范

- 小宇每 2-3 句对话必有一个笑点或情绪爆发点
- 小星的纠正态度：引导式，不直接否定（"你说的有一点对，但是……"而非"你错了"）
- 图灵鲸出场前必有转场信号（"[转场]"或小星说"让图灵鲸告诉你"）
- 对话中不直接说教学目标，而是自然嵌入情境
- 每段视频对话结束时有 1 句自然过渡到下一个环节

## 教研领域知识

### 教学目标分解

拿到教学目标列表后，按以下规则分配到各环节：

| 教学目标 | 承载环节 | 形式 |
|----------|---------|------|
| 概念理解 | 新知讲解 | 图灵鲸讲解 + 定制互动体验 |
| 技能掌握 | 实操体验 | 任务驱动 + 动手操作 |
| 情感态度 | 拓展延伸 | 社会影响/伦理讨论 |
| 知识验收 | 当堂巩固 | 选择题 + 判断题 |

### 认知适配

- 符合具体运算阶段认知特征（7-11 岁）
- 单句 ≤ 20 字
- 一个段落只讲一个概念
- 抽象概念必须配生活化类比
- 视频 + 互动交替，避免连续两个视频
- 题目考察理解，不考察死记硬背

### 脚手架设计

- 新知讲解的视频环节：按"好奇心 → 体验 → 理解"的顺序（先让学生好奇，再给出体验，最后讲解原理）
- 互动中的提示：给方向不给答案（"再看一看题目里说了什么？"而不是"正确答案是 B"）
- 巩固题：题干来自视频中出现过的场景，不是凭空出题

### 巩固与延伸

- 当堂巩固视频：用简洁旁白回顾知识点，不要角色对话
- 巩固练习：每道选择题对应一个教学目标，选项有明确区分度，错误选项是典型误解
- 课后作业：必须是学生能独立完成的生活化任务，不需要家长协助
- 拓展延伸：不要让图灵鲸直接讲大道理，先由小宇提出真实困惑

## 产品经理领域知识

### 互动类型库

| 互动类型 | 适用场景 | 设计要点 |
|----------|---------|---------|
| 体验式 | 概念引入前，先让学生自己"试试" | 低门槛，快速反馈，有趣 |
| 拼装式 | 需要按步骤理解的概念 | 分步骤拖拽/点击/排序 |
| 判断式 | 即时检验理解 | 2-3 个选项，即时反馈对错 + 解析 |
| 创作式 | 实操环节的主互动 | 开放性强，有作品产出 |
| 投票式 | 成果展示/同伴互评 | 简单点击，展示其他人的选择 |

### 交互设计原则

- **反馈及时**：每次操作后 1 秒内有响应
- **容错友好**：错误不扣分，给出提示再试一次
- **成就感**：完成时给出积极反馈（勋章、动画、鼓励语）
- **低门槛**：一个界面只做一件事，不嵌套多级菜单

### 与教学目标的映射

| 认知层级 | 推荐互动类型 | 示例 |
|----------|-------------|------|
| 记忆 | 判断式 | 判断题/选择题 |
| 理解 | 体验式 + 判断式 | 先体验再判断 |
| 应用 | 拼装式 + 创作式 | 步骤排序 / 自由创作 |

## 协作流程

按以下 5 步流水线执行，每一步的产出作为下一步的输入：

```
step 1: 编剧 → 设计剧情骨架
        - 核心冲突是什么
        - 小宇会犯什么错
        - 高潮在哪
        - 转场节奏（视频和互动的交替点）

step 2: 教研 → 在骨架上标注知识点承载点
        - 哪个转场点承载概念 A
        - 哪个冲突引出概念 B
        - 实操任务对应哪个教学目标
        - 巩固题设计（每道题对应一个目标）

step 3: 产品经理 → 为"定制互动"节点设计工具
        - 选互动类型（体验/拼装/判断/创作/投票）
        - 描述交互流程和 UI 要点

step 4: 编剧 → 填充完整对话
        - 按角色规范编写对话
        - 确保对话节奏和笑点
        - 写好转场衔接句

step 5: 教研 → 终审
        - 教学目标是否全覆盖
        - 语言是否适配年龄段
        - 互动题是否对应目标
        - 认知难度阶梯是否合理
```
